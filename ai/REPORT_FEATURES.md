# 결과보고서 및 통계 다운로드 운영 가이드

최종 업데이트: 2026-06-24  
운영 기준: v0.35

## 1. 기능 범위

영중폼 결과보고서는 외부 AI 없이 설문 응답 통계와 규칙 기반 문장을 조합해 생성합니다. 관리자는 자동 생성 문구를 직접 수정하고 저장한 뒤 Word 또는 PDF로 활용할 수 있습니다.

원본 응답, 원본 자유의견, CSV는 보고서 편집과 프로그램명 정규화의 영향을 받지 않습니다.

## 2. 결과보고서 생성

경로: `/admin/surveys/{surveyId}/responses`

1. 응답 관리 화면에서 응답과 분석 결과를 확인합니다.
2. `결과보고서 생성`을 누릅니다.
3. 설정 모달의 기본값을 확인하고 필요하면 수정합니다.
4. `보고서 열기`를 누릅니다.
5. 새 탭에서 `/admin/surveys/{surveyId}/report`가 열립니다.

### 설정 기본값

| 항목 | 기본값 |
| --- | --- |
| 보고서 제목 | `{설문명} 결과보고서` |
| 조사기간 시작일 | 최초 응답일 |
| 조사기간 종료일 | 최근 응답일 |
| 조사대상 | 해당 설문 응답자 |
| 작성부서 | 영중종합사회복지관 |
| 작성일 | 오늘 날짜 |
| 작성자 | 로그인 사용자 이름 또는 빈 값 |

설정값이 없거나 전달되지 않으면 보고서 페이지에서 안전한 기본값을 다시 계산합니다.

## 3. 보고서 구성

분석 데이터 존재 여부에 따라 일부 섹션 번호는 자동 조정됩니다.

1. 표지
2. 목차
3. 조사 개요
4. 응답자 특성
5. 만족도 분석
6. 자유의견
7. 종합 요약 및 개선방향

### 표지

- 기관명
- 보고서 제목
- 조사기간
- 조사대상
- 작성부서
- 작성일

### 조사 개요

- 설정 모달에서 입력한 기본 정보
- 설문 설명
- 관리자가 편집·저장한 조사 개요 해석문

### 응답자 특성

- 프로그램별 응답 현황
- 지역별 응답 현황
- 참여기간별 응답 현황
- 관리자가 편집 가능한 해석문

프로그램별 표는 보고서 길이를 줄이기 위해 상위 15개와 `기타` 합산 행을 사용합니다.

### 만족도 분석

- 전체 평균
- 응답자 수와 문항 응답 수
- 문항별 평균과 응답 수
- 상위·하위 문항
- 관리자가 편집 가능한 해석문

### 자유의견

- 자유의견 주요 유형
- 유형별 건수와 분석 문장
- 대표 의견
- 원본 자유의견 번호 목록
- 관리자가 편집 가능한 요약문

원본 자유의견은 수정하지 않습니다.

### 종합 요약 및 개선방향

- 응답 규모
- 전체 만족도 수준
- 상위·하위 문항의 의미
- 자유의견 주요 흐름
- 종합 평가
- 규칙 기반 개선방향

## 4. 자유의견 규칙 기반 분석

현재 분류체계:

- 긍정 평가
- 실천 적용 제안
- 교육 확대 요청
- 운영 개선 의견
- 홍보 및 참여 확대
- 시설 및 환경 의견
- 기타

### 분류 원칙

- 한 의견에는 최대 2개 유형을 부여합니다.
- `AI`, `교육`, `프로그램`, `강의` 같은 단어 하나만으로 요청 유형을 결정하지 않습니다.
- 시설·기자재 문맥은 교육 확대 요청과 중복 분류하지 않도록 우선 처리합니다.
- 원본 의견은 저장하거나 변경하지 않고 화면에서 계산합니다.

### 대표 의견 원칙

- 같은 원문은 대표 의견 영역에서 한 번만 사용합니다.
- 구체적이고 긴 문장과 제안 표현을 우선합니다.
- 단순 만족 표현과 단일 명사형 응답은 제외합니다.
- 대표 의견은 카테고리별 최대 3개를 표시합니다.

대표 의견에서 제외되는 예:

- 없음
- `-`
- 네
- 아니요
- 좋음
- 만족
- 매우 만족
- 감사합니다
- AI
- 연극
- 탁구
- 원예

원본 자유의견 목록에는 위 응답도 그대로 유지됩니다.

## 5. 보고서 편집과 저장

### 읽기 모드

- 관리자 화면으로 돌아가기
- 보고서 수정
- Word 다운로드
- 인쇄/PDF 저장

### 편집 모드

- 수정 취소
- 저장
- 저장 후 인쇄/PDF

편집 가능한 항목:

- 조사 개요 해석문
- 응답자 특성 해석문
- 만족도 분석 해석문
- 자유의견 요약문
- 향후 개선방향
- 종합 요약

표, 평균점수, 응답 수, 원본 자유의견은 편집할 수 없습니다.

### 저장 우선순위

1. `survey_reports/{reportId}`의 저장된 `sections`
2. 저장본이 없으면 현재 응답 분석으로 생성한 기본 문구

새로고침 후에도 저장 문구가 유지되며 Word와 PDF에도 저장본이 반영됩니다.

### 저장 문서 구조

기본 보고서는 `survey_reports/{surveyId}`에 저장됩니다. 복제본은 자동 생성 문서 ID를 사용합니다.

```text
survey_reports/{reportId}
  surveyId
  title
  periodStart
  periodEnd
  period
  target
  department
  author
  reportDate
  status: draft | final
  copiedFromReportId
  sections:
    overviewText
    respondentProfileText
    satisfactionAnalysisText
    openEndedSummaryText
    improvementPlanText
    finalSummaryText
  createdBy
  updatedBy
  createdAt
  updatedAt
  deleted
  deletedAt
  deletedBy
```

`deletedAt`, `deletedBy`, `copiedFromReportId`는 해당 작업이 있을 때만 존재할 수 있습니다.

## 6. Word 다운로드

Word 문서는 프론트엔드에서 `docx`와 `file-saver`로 생성합니다.

포함 내용:

- 표지
- 목차
- 조사 개요
- 응답자 특성
- 만족도 분석
- 자유의견 주요 유형
- 자유의견 원문
- 종합 요약 및 개선방향

### 출력 특성

- A4 기준 여백
- 제목과 소제목 스타일
- 회색 배경, 굵은 글씨, 가운데 정렬을 적용한 표 헤더
- Word에서 수정 가능한 표
- 자유의견 자동 번호 목록
- 저장된 보고문 우선 반영

파일명은 보고서 제목을 파일명에 안전한 형태로 변환해 `.docx`로 저장합니다.

## 7. PDF 저장

PDF는 별도 서버 변환이 아니라 브라우저 인쇄 기능을 사용합니다.

1. 읽기 모드에서 `인쇄/PDF 저장`을 누릅니다.
2. A4와 PDF 저장을 선택합니다.
3. 브라우저의 머리글과 바닥글을 해제합니다.
4. 미리보기에서 페이지 나눔을 확인합니다.

인쇄 시 다음 UI는 숨겨집니다.

- 보고서 상단 조작 버튼
- 저장 상태
- 편집 안내
- PDF 설정 안내
- 편집 가능한 영역 표시

표지와 목차는 독립 페이지로 구성되며, 반복 고정 footer는 출력하지 않습니다.

## 8. 결과보고서 관리

경로: `/admin/reports`

### 목록 정보

- 보고서 제목
- 원 설문명
- 응답 수
- 작성자
- 최종 수정일
- 상태

### 작업

| 작업 | 동작 |
| --- | --- |
| 열기 | 해당 `reportId`를 포함해 보고서 페이지를 엽니다. |
| Word 다운로드 | 저장본과 현재 분석 데이터를 조합해 Word를 생성합니다. |
| PDF 출력 | 보고서 페이지를 `print=1` 모드로 엽니다. |
| 복제 | 제목 앞에 `(복사본)`을 붙이고 새 `draft` 문서를 생성합니다. |
| 삭제 | 실제 삭제 없이 `deleted: true`로 처리합니다. |

목록은 삭제되지 않은 보고서만 표시하며 최종 수정일 최신순으로 정렬합니다.

## 9. 통계 Excel

경로: `/admin/surveys/{surveyId}/responses`

`통계 Excel 다운로드`를 누르면 `exceljs`를 사용해 브라우저에서 `.xlsx`를 생성합니다.

| 시트 | 주요 열 |
| --- | --- |
| 설문 개요 | 설문명, 응답 수, 응답 기간, 생성일 |
| 응답 원본 | 기존 CSV와 동일한 질문 및 응답 열 |
| 객관식 빈도분석 | 문항, 선택지, 응답 수, 비율, 문항 응답자 수 |
| 만족도 분석 | 문항, 평균, 응답 수, 표준편차, 최고점, 최저점 |
| 응답자 특성 | 프로그램, 지역, 참여기간 빈도 |
| 자유의견 분석 | 유형, 건수, 비율, 분석 결과, 대표 의견 |
| 종합요약 | 규칙 기반 보고서 자동 요약 |

### 데이터 보존 원칙

- Excel 생성은 Firestore 데이터를 수정하지 않습니다.
- `응답 원본` 시트는 CSV와 같은 열 순서를 사용합니다.
- 프로그램명 정규화는 분석 집계에만 적용합니다.
- 개인정보 포함 설문은 다운로드 전 확인 절차를 거칩니다.

## 10. 보고서 관련 감사로그

주요 action:

- `report_settings_opened`
- `report_opened`
- `report_edit_started`
- `report_saved`
- `report_summary_regenerated`
- `report_docx_downloaded`
- `report_print_clicked`
- `report_back_clicked`
- `report_unsaved_print_attempt`
- `report_list_opened`
- `report_opened_from_list`
- `report_word_downloaded_from_list`
- `report_copied`
- `report_deleted`
- `statistics_excel_downloaded`

감사로그에는 자유의견 원문 전체나 보고서 본문 전체를 저장하지 않습니다.

## 11. 운영 점검 체크리스트

- 설정 모달의 조사기간이 최초·최근 응답일과 일치하는가
- 보고서 저장 후 새로고침해도 문장이 유지되는가
- 읽기·편집 모드 버튼이 올바르게 전환되는가
- Word에 저장 문구와 편집 가능한 표가 들어가는가
- PDF에 관리자 UI와 브라우저 머리글·바닥글이 없는가
- 자유의견 대표 의견에 단순 응답이 노출되지 않는가
- 원본 자유의견과 CSV가 변경되지 않았는가
- 보고서 복제본이 별도 `reportId`로 열리는가
- soft delete 보고서가 목록에서 사라지는가
- 관련 감사로그 action이 기록되는가

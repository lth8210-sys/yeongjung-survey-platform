# QA Checklist

운영 배포 전에는 이 체크리스트를 기준으로 회귀 테스트를 수행한다. 확인한 항목은 체크하고, 미확인 항목은 사유를 적는다.

## Regression Matrix

| 이번 수정 | 영향 가능 | 확인 여부 |
| --- | --- | --- |
| Firestore rules | 설문 목록, 최근응답, 보고서, 템플릿 | [ ] |
| responses query | 공개 응답, 설문별 응답, 최근응답, CSV | [ ] |
| survey_reports query | 결과보고서 목록, 저장, DOCX | [ ] |
| quota 로직 | 응답 제출, quotaCounts, 통계 | [ ] |
| 조건분기 | 필수 응답, 페이지 흐름, 제출 | [ ] |
| 통계 계산 | 응답 관리, Excel, 결과보고서 | [ ] |

| logger 변경 | DEV path, PROD 마스킹 | [ ] |

## 2026-07 "2026 영중 지역주민 욕구조사" 개편 회귀 항목

이번 릴리즈에서 formTemplates.js와 quota 시스템이 크게 바뀌었으므로 아래를 추가로 확인한다.

- [ ] Q1이 주소 자유 입력형으로 정상 표시되고 저장되는지 확인
- [ ] Q45에서 영유아만 선택 시 Q46-1만 표시되는지 확인
- [ ] Q45에서 아동+청소년 선택 시 두 문항만 표시되는지 확인
- [ ] Q45에서 노년 선택 시 출생연도와 노년 욕구 문항이 함께 표시되는지 확인
- [ ] Q45 미선택 시 Q46 계열 문항이 전혀 표시되지 않는지 확인
- [ ] 조건부로 숨겨진 문항이 진행률 계산과 제출 payload에서 제외되는지 확인
- [ ] quota 대시보드가 연령대만 표시하고 지역 관련 UI가 없는지 확인
- [ ] CSV에 "주소" 컬럼이 있고 "권역"/"거주지역" 컬럼이 없는지 확인
- [ ] 결과보고서 생성 시 흰 화면 없이 정상 렌더되거나, 실패 시 원인 파악 가능한 안내 메시지가 뜨는지 확인
- [ ] 라이브 Firestore 설문 문서(`surveys/BUUry9Qt5sYZgDaZ8dSL`)는 2026-07-11 새 구조로
      반영 완료되었다 — 공개 응답 화면(`/surveys/BUUry9Qt5sYZgDaZ8dSL`)에서 구설문이 아닌
      새 설문(Q1 주소 자유입력 등)이 보이는지 확인. 동일 제목의 다른 published 문서
      (`DDNMRm97DGGRIRBEU4a8`, 응답 0건)가 목록에 함께 노출되면 혼동 소지 — 운영진과
      정리 필요 여부 공유
- [ ] (오탐 방지) `/admin/surveys/:id/preview` 미리보기는 정적 렌더링이며 조건부 분기/표시
      (branching, `visibilityConditions`)를 시뮬레이션하지 않는다 — Q46-1~6이 미리보기에서는
      모두 한꺼번에 보이는 것이 정상이다(기존 Q29/Q30 분기도 이미 동일하게 동작함). 조건부
      표시는 반드시 공개 응답 화면(`/surveys/:id`)에서 실제로 Q45를 선택해 확인한다.
- [ ] 새로 추가한 "경품 연락처"/"개인정보 수집·이용 동의" 문항 두 개를 체크하지 않아도
      설문 제출이 정상적으로 완료되는지 확인(둘 다 required: false)
- [ ] 조건부 개인정보 동의 정책 3가지 시나리오 확인: (1) 연락처 미입력+동의 미체크 →
      제출 가능 (2) 연락처 입력+동의 체크 → 제출 가능 (3) 연락처 입력+동의 미체크 →
      "연락처를 입력하신 경우 개인정보 수집 및 이용에 동의해주세요." 안내와 함께 제출 차단
- [ ] (KI-015 회귀) Q4에 정상적인 4자리 출생연도(예: 1974)를 입력하고 1페이지에서
      다음 페이지로 정상 진행되는지 확인 — quota 사용 설문에서 반드시 확인
- [ ] Q35가 복수선택(체크박스) 문항으로 표시되는지, Q34=아니오 선택 후 Q35 응답 시
      Q41로 정상 이동하는지 확인
- [ ] (KI-016 회귀) Survey Builder에서 "템플릿 적용"으로 새 설문을 만든 뒤(문항 ID가
      전부 재발급된 상태) 반드시 실제 공개 응답 화면에서 확인: (1) Q45 선택에 따라
      Q46-1~6이 조건부로 표시되는지 (2) 연락처 입력+동의 미체크 시 제출이 차단되는지.
      `/admin/surveys/:id/preview`나 문항 ID가 원본 그대로인 설문에서만 확인하면
      이 회귀를 놓친다 — 반드시 "템플릿 적용"을 거쳐 새로 생성된 설문으로 확인한다.

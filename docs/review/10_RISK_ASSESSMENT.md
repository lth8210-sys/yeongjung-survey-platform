# 10. Risk Assessment — 위험 종합 등록부

> 전 문서의 리스크를 단일 등록부로 집계. 위험도: **Critical / High / Medium / Low**.
> 위험도 = (영향도 × 발생가능성). 각 리스크에 소유 문서·완화책·우선순위 명시.

---

## 10.3 특별 검토 항목별 위험 요약

| 특별 검토 질문 | 판정 | 핵심 리스크 |
|----------------|------|-------------|
| 외부 응답자가 내부 데이터 접근 불가? | 읽기 ✅ / 쓰기 ⚠️ | RA-01 (공개 create 무방비) |
| 내부 직원 role/status 안정 작동? | 🔶 겨우 일치 | RA-07, RA-08 |
| 템플릿과 응답 데이터 분리? | ✅ 분리됨 | RA-16 (버전 경계만 없음) |
| 통계가 보고서에 도움? | ✅ 초안엔 유용 | RA-17 (확장성) |
| 모바일에서 주민 쉽게 응답? | 🔶 반응형 O·접근성 X | RA-18 |
| 문항 많아도 성능 유지? | 🔶 클라 감내·통계 비용↑ | RA-17 |
| 실수 삭제/수정 복구 가능? | 🔶 소프트삭제 O·백업 X | RA-05 |
| Firestore 비용 폭증 위험? | ❌ 위험 큼 | RA-01, RA-02, RA-13, RA-17 |

## 10.4 배포 게이팅 권고

**아래 Critical/보안 High는 규모 확대(다직원·대량 응답·외부 홍보) 전 반드시 완화:**
- RA-01, RA-02 (App Check + create 하드닝)
- RA-03 (CSV 인젝션)
- RA-05 (백업/PITR)
- RA-06, RA-09, RA-12 (PII)
- RA-14 (최소 회귀 테스트)

## 7.1 통계 엔진 (buildSurveyAnalytics)

### 현재 상태
`surveyAnalytics.js`(814줄)가 응답 배열을 받아:
- **점수 문항 자동 판별**: 척도 유형 + "1./2./..." 패턴의 단일선택을 점수화(`getNumericScore`).
- 문항별 평균·분포, 전체 평균(`totalAverage`), 상·하위 문항(`topRows`/`lowRows`).
- **자유응답 분류**(`classifyFreeTextAnswer`)·카테고리 요약(`buildFreeTextCategorySummary`).
- **프로그램명 정규화 집계**(`normalizeProgramName`) — 표기 흔들림을 묶어 집계.

### 강점
- 복지 현장 보고서 문법("총 N명 참여", 만족도 수준 문장, 강점/개선영역)을 규칙기반으로 자동 생성 → **실무 문서 초안 시간 절감**. 이건 이 플랫폼의 차별점.

### 문제점
- **점수 판별이 휴리스틱**: `/^([1-5])\./` 정규식으로 "1. 매우그렇다" 같은 표기에 의존. 라벨 표기가 다르면(예: "① 매우 만족") 점수화 실패 → 통계 누락. 문항 정의에 **명시적 점수 스키마가 없다.**
- **전량 메모리 계산**: 응답 전체를 클라이언트로 읽어 계산(05.3). 대량 응답에서 비용·지연.
- **분모 불명확 위험**: 분기(branching)로 특정 문항을 안 본 응답자와 무응답이 통계에서 같게 처리되면 평균이 왜곡. `skippedQuestionIds`/`visibleQuestionIds`가 응답에 저장되어 있으나(강점), 통계가 이를 분모에서 구분하는지 명시적 검증 필요.

### 위험도: Medium
### 개선 방향
- 문항에 **명시적 점수 매핑**(옵션별 value) 도입 → 정규식 추측 제거.
- 증분 집계로 O(1) 통계(05.3).
- 통계에 "응답 N / 노출 N / 무응답 N"을 분리 표기.

### 우선순위: P2(점수스키마 P1로 승격 가능) / 난이도: 중

## 7.2 결과보고서 자동화 (survey_reports + reportDocx)

### 현재 상태
- `generateRuleBasedReportSummary`·`generateRuleBasedImprovementPlan`이 통계로부터 요약·개선계획 문장 자동 생성.
- `survey_reports`에 섹션별 텍스트 저장(draft/final), 복사(`copySurveyReport`), 소프트삭제.
- DOCX 생성(`reportDocx.js`, 574줄, `docx` 라이브러리) — 기간/대상/부서/작성자/섹션 본문 포함.

### 강점 — 보고서 자동화 기반이 이미 있다
"통계 → 문장 초안 → 편집 → DOCX 저장·다운로드" 파이프라인이 동작. v2의 핵심 자산으로 계승.

### 문제점
- **규칙기반 문장의 경직성**: 만족도 4점 이상/미만 두 갈래 등 템플릿 문장. 조사 성격(사정/욕구/만족도)에 따른 서술 다양성 부족 → 보고서가 비슷해 보임.
- **시각자료 부재**: DOCX가 텍스트 표 위주. 문항별 분포 **차트 이미지**가 없어 보고서 품질이 수기 그래프에 의존.
- **보고서 ↔ 통계 시점 불일치**: 보고서 저장 후 응답이 더 들어오면 저장된 요약과 실시간 통계가 어긋남. `periodStart/End`는 있으나 **통계 스냅샷을 보고서에 고정 저장**하는 구조가 아님(sections 텍스트만 저장).
- LLM 요약(더 자연스러운 서술) 여지 — 현재 없음. 규칙기반만.

### 위험도: Medium
### 개선 방향
- 보고서 저장 시 **통계 스냅샷(JSON)을 함께 저장** → 재현성·시점 일관성.
- 문항별 차트를 SVG/PNG로 렌더 → DOCX 삽입.
- (선택) 조사 유형별 서술 템플릿 확장, 필요 시 Claude API 요약을 초안 보조로.

### 우선순위: P2 / 난이도: 중

## 7.3 엑셀 / CSV 내보내기

### 현재 상태
- CSV: `downloadCsv` + `escapeCsvValue`(응답원본/명단형/슬롯 3종). BOM(`﻿`) 부착 → 한글 엑셀 호환.
- Excel: `statisticsExcel.js`(473줄, `exceljs`) — 통계 워크북 생성.
- 다운로드 시 마스킹(`maskResponsesForDownload`) + 감사로그(`responses_csv_downloaded`).

### 문제점 — **CSV 인젝션 (Critical 후보)**
`escapeCsvValue`는 큰따옴표만 이스케이프(`"` → `""`)하고 값을 `"..."`로 감쌈. 그러나 **`=`, `+`, `-`, `@`로 시작하는 셀 값(수식 인젝션)을 무력화하지 않는다.**
- 응답자가 이름/자유응답에 `=HYPERLINK("http://evil","click")` 또는 `=cmd|...` 같은 값을 넣으면, 관리자가 CSV를 엑셀로 열 때 **수식이 실행**되어 데이터 유출·명령 실행 위험.
- 복지관 CSV는 직원 PC에서 열림 → 직접적 사내 위협.

### 위험도: **High** (CSV 수식 인젝션)
### 개선 방향
- 셀 값이 `= + - @ \t \r`로 시작하면 앞에 작은따옴표(`'`) 또는 공백 prefix.
- exceljs 출력에도 동일 sanitize 적용.

### 기타 문제점
- 마스킹이 **다운로드 시 선택 파라미터**(`piiQuestionIds` 없으면 원본 그대로). 원본 CSV(`survey.title-응답원본.csv`)는 PII 포함 다운로드 경로가 존재 → 권한(`canDownloadResponses` = creator 이상)만으로 원본 반출 가능. 반출 로그는 있으나 마스킹 강제는 아님.

### 위험도: Medium (PII 원본 반출)
### 개선 방향: 원본 반출은 admin 한정 + 사유 입력 + 강제 감사. creator는 마스킹본만.
### 우선순위: P1(CSV 인젝션) / P2(원본 반출 통제) / 난이도: 낮(인젝션)~중

## 7.4 통계 화면이 보고서 작성에 도움 되는가 (특별 검토)

### 결론: **초안 생성엔 확실히 도움. 최종 품질은 수작업 의존.**
- 도움 되는 점: 참여자 수·평균·상하위·자유응답 키워드·프로그램별 집계 자동 산출 → 보고서 뼈대 즉시 확보.
- 부족한 점: 차트 이미지·시점 스냅샷·교차분석(예: 연령대×만족도)·통계적 유의성 부재. 심층 보고서는 여전히 엑셀로 별도 작업 필요.

### 개선 방향(요약): 스냅샷 저장 + 차트 export + 교차표(피벗) 1종 추가로 "붙여쓸 수 있는 보고서"로 승격.
### 우선순위: P2 / 난이도: 중
